脅威モデルは、一般論の不安を、具体的な攻撃経路、信頼境界分析、緩和策の選択へ落とし込むための道具です。

## 概要

アクセス制御の失敗は単一バグとして現れることは稀です。多くは連鎖です。たとえば古い権限と弱いレビュー、トークン窃取と分離不足、プロンプトインジェクションと広すぎるツールアクセス、サービスアカウントの乱立と属性把握不足の組み合わせです。脅威カタログは、インシデント前にこれらを議論する共通言語を提供します。

## 主要概念

### 代表的な脅威カテゴリ

- **権限昇格**: 誤設定、ロール継承、欠陥のあるポリシー評価による昇格
- **トークン / 資格情報の窃取**: エンドポイント、ワークロード、CI/CD、ログからの窃取
- **横方向移動**: 広すぎるサービス信頼や共有資格情報を使った横展開
- **孤立アカウント**: 所有者消失後も残るアカウント
- **影の管理権限**: 間接権限や統制外サポートツールによる隠れた管理権限
- **代理人の混乱問題**: 信頼されたサービスが他者の意図で権限を誤用する
- **プロンプトインジェクション**: エージェントのツール利用やデータ解釈に影響する入力
- **ポリシー回避**: 文書化されていない例外経路や層ごとの適用不一致

### 脅威モデルの構造

各脅威について、少なくとも次を整理します。

1. 攻撃者の前提条件
2. 越えられる信頼境界
3. 危険に晒される資産
4. 検知機会
5. 予防的制御と補完的制御

## 実装と運用

### 緩和策の例

- 古い権限 -> ライフサイクル自動化、アクセスレビュー、所有者対応付け
- トークン窃取 -> 短命資格情報、デバイス束縛、保管庫の衛生管理、異常検知
- 横方向移動 -> 分離、最小権限、サービスアイデンティティの分離
- プロンプトインジェクション -> ツール許可リスト、コンテンツ分離、承認ゲート、出力フィルタリング
- ポリシー回避 -> 統一されたポリシー意味論、否定系テスト、一貫したログ

脅威モデルが最も有効なのは、静的なコンプライアンス資料として保存されるときではなく、アーキテクチャレビュー、制御テスト、事後学習に結び付いているときです。
