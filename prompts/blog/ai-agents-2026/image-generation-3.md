# Image Generation Prompt — Figure 3

Create a publication-quality layered architecture diagram image titled:

"The New Agent Stack"

Subtitle:

"How agent systems evolved from models into layered operating platforms"

Style:

- architecture reference model
- enterprise architecture whitepaper quality
- clean vector graphics
- white background
- minimalist but information-rich
- 16:9 aspect ratio
- suitable for a technical blog
- emphasize architecture and relationships rather than products

Central message:

Agent systems are becoming layered operating platforms rather than collections of prompts and scripts.

The diagram should clearly communicate separation of concerns between:

- models
- enterprise capabilities
- interoperability
- runtime execution
- managed platforms
- applications

Create a vertically layered architecture stack.

Layer 1: Applications

Examples:

- Coding Agents
- Data Agents
- Research Agents
- Support Agents
- Operations Agents

Caption:

"Business-facing agent experiences"

Layer 2: Managed Agent Platforms

Capabilities:

- Deployment
- Lifecycle Management
- Governance Integration
- Enterprise Packaging
- Shared Services

Caption:

"Platforms package and operate the layers below"

Agent Infrastructure (Agent Runtime & Orchestration, Protocol Layer, Tools & Data, Models)

Layer 3: Agent Runtime & Orchestration

Capabilities:

- Workflows
- State Management
- Multi-Step Execution
- Human Approvals
- Retries
- Scheduling

Caption:

"Runtime control for agent behavior"

Layer 4: Protocol Layer

Examples:

- MCP
- A2A
- ACP
- Emerging Interoperability Contracts

Caption:

"The interoperability layer"

Layer 5: Tools & Data

Examples:

- APIs

- Event Streams

Caption:

"Enterprise capabilities exposed to agents"

Layer 6: Models

Examples:

- Reasoning Models
- Multimodal Models
- Coding Models

Caption:

"The intelligence substrate"

Visual Guidance:

- The layered stack should occupy most of the visual space.
- Cross-cutting controls should appear secondary.
- Use architecture icons rather than vendor logos.
- Avoid marketplace or vendor-landscape aesthetics.
- The overall feel should resemble a cloud-native reference architecture.
