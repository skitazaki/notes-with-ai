# Image Generation Prompt - Ownership Boundaries Example

Create a documentation-quality example figure titled:

"Example: Platform and Product-Team Ownership"

Purpose:

Illustrate a modern system where product teams, a platform team, security, and data ownership all coexist over the same architecture.

Style:

- organizational-technical architecture graphic
- clean vector overlay design
- light background
- clear ownership regions without visual clutter
- 16:9 aspect ratio

Composition:

Show a shared platform-based system with components such as:

- domain applications
- deployment and runtime platform
- identity and policy services
- observability and secrets management
- event or data model layer

Overlay ownership boundaries or responsibility bands for:

- product teams
- platform team
- security team
- data team

Include a few explicit contract or handoff points, such as schema approval, policy standards, or incident escalation.

Central message:

Ownership boundaries overlap with technical structure, but they are not the same thing.

Do:

- show shared capability ownership clearly
- emphasize accountability for change and operation
- keep the example realistic for a platform plus product-team model

Do not:

- force a one-service-one-team mapping everywhere
- turn the graphic into a corporate org chart
- imply that shared platform capabilities are ownerless
