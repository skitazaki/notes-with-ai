# Image Generation Prompt - Ownership Boundaries Example

Create a clean architecture example diagram titled:

"Example: Platform and Product-Team Ownership"

Use case:

A company runs several product applications on a shared internal platform.
Product teams own customer-facing services, the platform team owns shared runtime capabilities, the security team owns policy and identity controls, and the data team owns shared event and analytics contracts.

Image content:

Show a shared platform architecture with components such as:

- storefront app
- billing service
- support service
- API gateway
- deployment platform
- runtime platform
- identity service
- policy service
- observability
- event stream
- analytics model

Overlay ownership regions or boundary outlines for:

- product teams
- platform team
- security team
- data team

Add compact handoff markers:

- service contract
- policy standard
- schema approval
- incident escalation

Style:

- precise technical architecture diagram
- clean vector overlay design
- light background
- restrained colors for ownership regions
- 16:9 aspect ratio
- documentation-ready

Text rules:

- Use short labels only.
- Do not include captions, takeaways, or explanatory notes.
- Avoid dense text blocks.

Do:

- show ownership boundaries over real platform components
- make shared capabilities visibly owned
- keep the example practical and vendor-neutral

Do not:

- show people, desks, screens, or room interiors
- turn the diagram into a corporate org chart
- force every component into exactly one owner
