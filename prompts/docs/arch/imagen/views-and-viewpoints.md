# Image Generation Prompt - Views and Viewpoints

Create a conceptual documentation figure titled:

"Views and Viewpoints"

Purpose:

Show that a single system can generate several useful architecture views for different stakeholders.
The image should emphasize selective communication rather than total completeness.

Style:

- editorial technical infographic
- clean vector illustration
- light background
- framed panels or windows
- precise labeling
- 16:9 aspect ratio

Composition:

Place one abstract platform in the center as the source system.
Derive 4 to 6 surrounding framed views for different audiences, such as:

- Executive View
- Developer View
- Operations View
- Security View
- Reliability View
- Data or Governance View

Each view should highlight different concerns from the same underlying system.
Make it visually clear that each frame hides irrelevant detail on purpose.

Central message:

An architecture view is a selective communication artifact.
Different stakeholders need different representations of the same system.

Include a subtle caption with this idea:

"Useful views are intentional, not exhaustive."

Do:

- show visual continuity between the source system and each derived view
- make audience differences clear at a glance
- keep the figure structured and uncluttered

Do not:

- create one giant master diagram
- make every view equally detailed
- depend on any specific modeling standard
