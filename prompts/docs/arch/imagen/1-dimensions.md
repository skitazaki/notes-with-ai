# Image Generation Prompt - Architecture Dimensions

Create a documentation-quality conceptual infographic titled:

"Architecture Dimensions"

Purpose:

Show that architecture dimensions are complementary lenses for reasoning about the same system.
The image should make it clear that each dimension highlights a different concern and hides different details.

Style:

- precise editorial infographic
- vector-style technical illustration
- light background
- calm, professional color palette
- clean labels and spacing
- 16:9 aspect ratio
- suitable for a conceptual docs page

Composition:

Place one abstract platform diagram in the center.
Project it outward into five surrounding panels or slices labeled:

- Structural
- Operational
- Strategic
- Ownership
- Communication

Show each panel with a distinct visual cue:

- Structural uses components and dependency lines
- Operational uses motion, routing, or execution paths
- Strategic uses quality markers or tradeoff badges
- Ownership uses grouped domains or responsibility zones
- Communication uses framed audience views

Add a subtle chain or footer sequence:

Concern -> Dimension -> Reasoning -> View

Central message:

Each dimension answers a different primary question.
No single representation captures all architecture concerns well.

Do:

- make the five dimensions feel equally important
- keep the running system consistent across all views
- optimize for clarity at article width

Do not:

- turn this into a taxonomy poster with dense text
- imply that dimensions are a maturity ladder
- use vendor branding
