# Image Generation Prompt - Layers

Create a publication-quality explanatory figure titled:

"Layers"

Purpose:

Illustrate layers as a structural abstraction for dependency direction, responsibility, and insulation from change.

Style:

- clean architecture documentation graphic
- crisp vector blocks and arrows
- light background
- restrained enterprise palette
- readable at laptop width
- 16:9 aspect ratio

Composition:

Show a modern platform as stacked architectural layers from bottom to top.
Use 5 labeled layers such as:

- Infrastructure
- Platform Services
- Runtime and Orchestration
- Application and Domain Logic
- User and Integration Interfaces

Show dependency direction clearly, mainly upward or inward, with a few controlled arrows.
Include small side annotations for:

- abstraction
- dependency flow
- change isolation

Central message:

Layers show what builds on what.
They are about structural abstraction, not runtime request flow or team ownership.

Optional visual contrast:

Add a faint side note indicating that deployment tiers and request paths are different concepts.

Do:

- make the stack easy to scan
- emphasize dependency structure over runtime behavior
- keep the example generic and vendor-neutral

Do not:

- draw a sequence diagram
- map every internal dependency
- present layers as the only valid architecture model
