# Image Generation Prompt - Planes

Create a technical documentation figure titled:

"Planes"

Purpose:

Show planes as operational slices that cut across the same platform structure.
The image should clearly distinguish planes from structural layers.

Style:

- modern enterprise architecture diagram
- vector illustration
- light background
- strong visual separation between structure and operation
- labeled cross-cutting overlays
- 16:9 aspect ratio

Composition:

Begin with a faint layered platform in the background.
Across it, overlay several crossing horizontal or diagonal operational planes labeled:

- Control Plane
- Data Plane
- Observability Plane
- Policy Plane
- Workflow Plane

Use each plane to indicate a different kind of runtime responsibility spanning multiple components.
Show a few clear callouts such as:

- controls execution
- processes traffic or data
- captures telemetry
- enforces policy
- orchestrates work

Central message:

Planes explain how responsibility operates across a system.
They are cross-cutting runtime slices, not dependency stacks.

Do:

- make the contrast with layers immediately visible
- keep labels concise and practical
- suggest runtime behavior without overloading the image

Do not:

- turn every subsystem into its own plane
- make the image look like a generic network topology map
- use vendor-specific icons
