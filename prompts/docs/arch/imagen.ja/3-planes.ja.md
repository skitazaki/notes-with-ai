# 画像生成プロンプト — プレーン

以下のタイトルを持つ、技術ドキュメント向けの図を作成してください。

"プレーン"

目的:

プレーンを、同じプラットフォーム構造を横断する運用上の切り口として示してください。
この図では、プレーンが構造レイヤーとは異なる概念であることを明確に区別してください。

スタイル:

- モダンなエンタープライズアーキテクチャ図
- ベクター図解
- 明るい背景
- 構造と運用の強い視覚的分離
- ラベル付きの横断オーバーレイ
- 16:9 のアスペクト比

構図:

背景に、うっすらとしたレイヤー型プラットフォームを配置してください。
その上に、複数の横断的な水平または斜めの運用プレーンを重ねてください。

- コントロールプレーン
- データプレーン
- オブザーバビリティプレーン
- ポリシープレーン
- ワークフロープレーン

各プレーンは、複数コンポーネントにまたがる異なる実行時責務を示すものにしてください。
次のようなコールアウトを少数加えてください。

- 実行を制御する
- トラフィックやデータを処理する
- テレメトリを収集する
- ポリシーを適用する
- 作業をオーケストレーションする

中心メッセージ:

プレーンは、責務がシステム全体でどのように機能するかを説明します。
それは依存スタックではなく、横断的な実行時スライスです。

含めること:

- レイヤーとの違いがすぐに見て取れること
- ラベルは簡潔かつ実務的であること
- 情報過多にせず実行時の振る舞いを示唆すること

避けること:

- すべてのサブシステムを独立したプレーンにすること
- 汎用的なネットワークトポロジ図に見せること
- ベンダー固有アイコンの使用
