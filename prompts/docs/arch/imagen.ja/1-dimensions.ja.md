# 画像生成プロンプト — アーキテクチャのディメンション

以下のタイトルを持つ、ドキュメント品質の概念インフォグラフィックを作成してください。

"アーキテクチャのディメンション"

目的:

アーキテクチャディメンションが、同じシステムを推論するための相補的なレンズであることを示してください。
各ディメンションが異なる関心事を強調し、異なる詳細を隠すことが明確に伝わるようにしてください。

スタイル:

- 精密な編集調インフォグラフィック
- ベクタースタイルの技術図解
- 明るい背景
- 落ち着いたプロフェッショナルな配色
- クリーンなラベルと余白
- 16:9 のアスペクト比
- 概念ページ向けの見た目

構図:

中央に 1 つの抽象的なプラットフォーム図を配置してください。
そこから外側に向かって、5 つの周辺パネルまたはスライスへ投影してください。

- 構造
- 運用
- 戦略
- オーナーシップ
- コミュニケーション

各パネルには、それぞれ異なる視覚的手がかりを与えてください。

- 構造はコンポーネントと依存線を使う
- 運用は動き、ルーティング、または実行経路を使う
- 戦略は品質マーカーやトレードオフのバッジを使う
- オーナーシップはグループ化されたドメインや責任領域を使う
- コミュニケーションは対象読者別のフレーム化されたビューを使う

控えめな連鎖またはフッターの並びを追加してください。

関心事 -> ディメンション -> 推論 -> ビュー

中心メッセージ:

各ディメンションは異なる主要な問いに答えます。
単一の表現だけで、すべてのアーキテクチャ上の関心事をうまく捉えることはできません。

含めること:

- 5 つのディメンションが同じくらい重要に感じられること
- すべてのビューで対象システムの一貫性を保つこと
- 記事幅での可読性を優先すること

避けること:

- 文字量の多い分類ポスターにしてしまうこと
- ディメンションを成熟度の段階のように見せること
- ベンダーブランディングの使用
