# 画像生成プロンプト — レイヤーの例

# タイトル

**例: レイヤーとして捉える開発者プラットフォーム**

---

# ユースケース

中規模のソフトウェア企業が、プロダクトチーム向けに内部開発者プラットフォームを運用しています。

このプラットフォームには、

- クラウド基盤
- 共有ランタイムサービス
- 開発者向けセルフサービス機能
- ドメインアプリケーション
- 外部連携チャネル

が含まれます。

この具体例を通して、レイヤーが構造を整理するのに有効であることを示してください。

---

# 構図

下から上へ、5 つの積層レイヤーを作成してください。

## レイヤー名

- クラウド基盤
- プラットフォームランタイムサービス
- 共有開発者機能
- ドメインアプリケーション
- ユーザーと連携チャネル

## 各レイヤーに入れる代表要素

- ネットワーク、コンピュート、ストレージ
- コンテナランタイム、シークレット、アイデンティティ、オブザーバビリティ
- CI/CD、サービスカタログ、内部ポータル、テンプレート
- 注文サービス、請求サービス、サポートサービス
- Web アプリ、モバイルアプリ、パートナー API

依存矢印は少数にとどめ、主に上向き、または内向きであることが伝わるようにしてください。

---

# 重要な見せ方

この図で強調したいのは、

- 何が基盤で
- 何がその上に成り立ち
- どこで共通機能が支えになっているか

という構造です。

実行時のリクエストフローではなく、構成と依存の秩序が見えるようにしてください。

---

# スタイル

- 精密な技術アーキテクチャ図
- クリーンなベクターブロック
- 明るい背景
- 落ち着いたエンタープライズ向け配色
- 16:9
- ドキュメント利用に適した見た目

---

# テキストのルール

- 短いコンポーネントラベルのみを使うこと
- キャプションや長い説明文を入れないこと
- すべてのラベルを読みやすく保つこと

---

# 必ず含めること

- 実在しそうな内部エンジニアリング基盤に見えること
- 構造的レイヤーと依存方向が明確であること
- ベンダー中立であること

---

# 避けること

- 人物、机、画面、室内風景を描くこと
- 実行時のリクエストフローを描くこと
- 装飾的な柱の比喩や製品ロゴを使うこと
