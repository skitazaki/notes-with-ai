# 画像生成プロンプト — レイヤーの例

以下のタイトルを持つ、クリーンなアーキテクチャ例示図を作成してください。

"例: レイヤーとして捉える開発者プラットフォーム"

ユースケース:

中規模のソフトウェア企業が、プロダクトチーム向けに内部開発者プラットフォームを運用しています。
このプラットフォームには、クラウド基盤、共有ランタイムサービス、開発者向けセルフサービス機能、ドメインアプリケーション、外部連携チャネルが含まれます。

画像の内容:

下から上へ、5 つの積層レイヤーを作成してください。

- クラウド基盤
- プラットフォームランタイムサービス
- 共有開発者機能
- ドメインアプリケーション
- ユーザーと連携チャネル

各レイヤーの中には、代表的な機能を配置してください。

- ネットワーク、コンピュート、ストレージ
- コンテナランタイム、シークレット、アイデンティティ、オブザーバビリティ
- CI/CD、サービスカタログ、内部ポータル、テンプレート
- 注文サービス、請求サービス、サポートサービス
- Web アプリ、モバイルアプリ、パートナー API

上向き、または内向きの依存矢印を少数だけ描いてください。

スタイル:

- 精密な技術アーキテクチャ図
- クリーンなベクターブロック
- 明るい背景
- 抑制されたエンタープライズ向け配色
- 16:9 のアスペクト比
- ドキュメント利用に適した見た目

テキストのルール:

- 短いコンポーネントラベルのみを使うこと
- まとめ、キャプション、説明注記を入れないこと
- すべてのラベルを読みやすく保つこと

含めること:

- 実在しそうな内部エンジニアリング基盤として見えること
- 構造的レイヤーと依存方向を強調すること
- 例はベンダー中立に保つこと

避けること:

- 人物、机、画面、室内風景を描くこと
- 実行時のリクエストフローを描くこと
- 装飾的な柱の比喩や製品ロゴを使うこと
