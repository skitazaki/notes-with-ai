# 画像生成プロンプト — オーナーシップ境界の例

以下のタイトルを持つ、クリーンなアーキテクチャ例示図を作成してください。

"例: プラットフォームとプロダクトチームのオーナーシップ"

ユースケース:

ある企業が、共有の内部プラットフォーム上で複数のプロダクトアプリケーションを運用しています。
プロダクトチームは顧客向けサービスを所有し、プラットフォームチームは共有ランタイム機能を所有し、セキュリティチームはポリシーとアイデンティティ制御を所有し、データチームは共有イベントと分析契約を所有しています。

画像の内容:

次のようなコンポーネントを持つ共有プラットフォームアーキテクチャを示してください。

- ストアフロントアプリ
- 請求サービス
- サポートサービス
- API ゲートウェイ
- デプロイメント基盤
- ランタイム基盤
- アイデンティティサービス
- ポリシーサービス
- オブザーバビリティ
- イベントストリーム
- 分析モデル

その上に、次の責任領域または境界アウトラインを重ねてください。

- プロダクトチーム
- プラットフォームチーム
- セキュリティチーム
- データチーム

コンパクトなハンドオフマーカーを加えてください。

- サービス契約
- ポリシー標準
- スキーマ承認
- インシデントエスカレーション

スタイル:

- 精密な技術アーキテクチャ図
- クリーンなベクターオーバーレイデザイン
- 明るい背景
- オーナーシップ領域ごとの抑制された配色
- 16:9 のアスペクト比
- ドキュメント利用に適した見た目

テキストのルール:

- 短いラベルのみを使うこと
- キャプション、要点、説明注記を入れないこと
- 密度の高いテキストブロックを避けること

含めること:

- 実在しそうなプラットフォームコンポーネントの上にオーナーシップ境界を示すこと
- 共有機能にも明確なオーナーがいると見えること
- 実務的でベンダー中立の例にすること

避けること:

- 人物、机、画面、室内風景を描くこと
- 図を企業の組織図にしてしまうこと
- すべてのコンポーネントを厳密に 1 人の所有者へ押し込めること
