# Convergence の日本語訳の修正

対象ファイル:

`content/docs/data/management/data-quality-dimensions/_index.ja.md`

"Convergence" は文脈によってかなり訳し分けられます。
**収束** は正しいのですが、少し数学的・物理学的な印象があります。
また「歴史的現象」の整理をしたい訳ではありません。
一般的な文章として自然なニュアンスに変更します。

設計上の整理を明確にしてください。

**「様々なフレームワークを比較すると、6つのコア観点を共通モデルとして抽出できる」**

# 集約や共通化

「収束」よりも集約や共通化の方が読み手には伝わりやすいと考えられる。

- フレームワークの共通化
- フレームワークの集約
- 共通モデルへの集約（セクションタイトルとしてはこれを採用）
- 中核観点への集約

ドキュメントの意図：

- DAMA、ISO、Wang & Strong、IBM、Atlan などは異なる出発点を持つが、結果的には Accuracy / Completeness / Consistency などの中核観点へ集まっている
- DAMA、ISO 25012、Wang & Strong などの主要な品質フレームワークは、表現や粒度の違いはあるものの、最終的には共通する中核観点へ集約されている。
